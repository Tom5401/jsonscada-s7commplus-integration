1. **Add two local variables** immediately before the `return new BsonDocument` statement (i.e., after line 207 / the `associatedValues` array, before line 209 / the `return`):

```csharp
uint relationId = (uint)(dai.CpuAlarmId >> 32);
uint dbNumber   = relationId & 0xFFFF;
```

Per D-01: `relationId` is extracted by right-shifting `CpuAlarmId` (a `ulong`) by 32 bits and casting to `uint`. This is the exact inverse of the packing in `BrowseAlarms.cs:414`.

Per D-02: `dbNumber` uses the **lower 16 bits** of `relationId` (not `>> 16`). Confirmed by `S7CommPlusConnection.cs:1247`.

2. **Add two new fields** to the `BsonDocument` literal, immediately after the `allStatesInfo` entry (currently line 226). Insert these two lines before the closing `};`:

```csharp
{ "relationId",        new BsonInt64((long)relationId) },
{ "dbNumber",          (int)dbNumber }
```

Per D-03: `relationId` MUST be `BsonInt64`, not `BsonInt32`. Values like `0x8a0e0005` (2,317,140,997) exceed `Int32.MaxValue` and would silently wrap to negative. The `(long)` cast from `uint` is safe (uint max 4,294,967,295 fits in long).

Per D-04: `dbNumber` is stored as `BsonInt32` via `(int)dbNumber`. DB numbers max at 65535 which fits in Int32.

Per D-05: Do NOT add `originDbName` — that is Phase 6 scope. Do NOT modify the function signature — it stays as `BuildAlarmDocument(AlarmsDai dai, S7CP_connection srv)`.
