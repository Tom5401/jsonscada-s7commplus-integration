<objective>
Fix the ackState always-true bug in BuildAlarmDocument() so MongoDB alarm documents correctly reflect PLC acknowledgement state.

Purpose: Every alarm event currently appears acknowledged in MongoDB because the ackState sentinel is wrong — `DateTime.MinValue` is compared against an `AckTimestamp` that the library initialises to Unix epoch (1970-01-01) for unacknowledged alarms, so the expression always evaluates to true. This plan confirms the correct sentinel via a live PLCSIM debugging session then applies a single-line surgical fix.

Output: `AlarmThread.cs` with corrected ackState expression and a confirming comment. MongoDB alarm documents thereafter reflect actual PLC ack state.
</objective>
