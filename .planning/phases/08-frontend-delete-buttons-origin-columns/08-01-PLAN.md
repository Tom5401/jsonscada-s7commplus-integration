### 3. Add delete handler functions (after `fetchAlarms`)

Add these four functions:

```javascript
const handleDeleteRow = (item) => {
  if (item.alarmState === 'Coming' && !item.ackState) {
    confirmState.value = { visible: true, type: 'row-active', item, count: 0 }
  } else {
    executeDeleteRow(item)
  }
}

const executeDeleteRow = async (item) => {
  const idx = alarms.value.findIndex(a => a._id === item._id)
  if (idx !== -1) alarms.value.splice(idx, 1)
  confirmState.value.visible = false
  await fetch('/Invoke/auth/deleteS7PlusAlarms', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ ids: [item._id] })
  })
}

const handleDeleteFiltered = () => {
  confirmState.value = { visible: true, type: 'bulk', item: null, count: filteredAlarms.value.length }
}

const executeDeleteFiltered = async () => {
  const ids = filteredAlarms.value.map(a => a._id)
  const toRemove = new Set(ids)
  alarms.value = alarms.value.filter(a => !toRemove.has(a._id))
  confirmState.value.visible = false
  await fetch('/Invoke/auth/deleteS7PlusAlarms', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ ids })
  })
}
```

CRITICAL: In `executeDeleteFiltered`, the `const ids = filteredAlarms.value.map(a => a._id)` line MUST come BEFORE the `alarms.value = alarms.value.filter(...)` line. After mutation, `filteredAlarms` recomputes and the IDs are gone.

Per D-06: `handleDeleteRow` fires `executeDeleteRow` immediately for normal rows (no dialog).
Per D-07: `handleDeleteRow` opens the `row-active` dialog for Coming+unacked rows.
Per D-08: `handleDeleteFiltered` always opens the `bulk` dialog.
Per D-09: No error handling on failed fetch — the 5s auto-refresh restores rows.
Per D-10: Optimistic removal uses `response` is not checked — removal happens before fetch.
Per D-04: No `Authorization` header — cookie auth matches existing `fetchAlarms` pattern.
