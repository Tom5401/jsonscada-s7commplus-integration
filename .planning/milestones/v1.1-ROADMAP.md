### Phase 4: Ack Write-Back

**Goal:** An operator can acknowledge an unacknowledged alarm from the viewer and the PLC receives and applies the acknowledgement command — closing the full ack loop from UI to PLC
**Depends on:** Phase 3
**Requirements:** DRVR-03, VIEW-03
**Plans:** 2 plans

Plans:
- [x] 04-01-PLAN.md — Wireshark spike: capture and decode alarm ack PDU format, prototype C# method
- [x] 04-02-PLAN.md — Full ack pipeline: Express endpoint + C# ack branch + Vue Ack button with pending state

**Details:**
Wireshark spike decoded AckJob as `CreateObjectRequest` (not `SetVariableRequest` as initially assumed) with 13 numeric IDs extracted from s7comm_plus.dll. `SendAlarmAck()` implemented on `S7CommPlusConnection` using `alarmConn` (alarm subscription connection — no polling contention). `POST /Invoke/auth/ackS7PlusAlarm` inserts to `commandsQueue` with `s7plus-alarm-ack` ASDU. C# change stream branch dequeues via `PendingAcks` queue to `AlarmThread`, updates `ackState` in `s7plusAlarmEvents`, and updates `commandsQueue` delivered/ack fields. Vue Ack button shows spinner during pending, resolves on next poll cycle. AckJob completion notification handling added to AlarmThread (`ClassId == 3636` skip).

## Milestone Summary

**Key Decisions:**
- DateTime.UnixEpoch (not DateTime.MinValue) is the correct ackState sentinel — confirmed via PLCSIM trace showing protocol sends 0 for unacknowledged alarms
- AlarmClass 33 = "Acknowledgment required" — seeded from PLCSIM trace; Unknown (N) fallback covers unmapped IDs
- AckJob uses CreateObjectRequest (not SetVariableRequest) — DynObjX7 dynamic RelId in notification response is the definitive marker; only CreateObject produces dynamic RelIds
- SendAlarmAck routed through alarmConn (alarm subscription connection) — avoids polling contention with tag connection
- AckJob completion notification (ClassId=3636) skipped in AlarmThread — NotImplementedException eliminated; AlarmsDai.FromNotificationObject would fail on AckJob attributes

**Issues Resolved:**
- ackState always-true bug (DateTime.MinValue was sentinel; should be DateTime.UnixEpoch)
- BsonDouble/System.Double serializer mismatch in rtCommand (removed BsonDoubleSerializer attribute from double fields)
- AlarmThread crash on AckJob completion notification (unknown PDU type)
- SendAlarmAck error=5 on receive (ack confirmation PDU unparseable — fixed by routing ack through alarmConn and skipping AckJob notification)

**Issues Deferred (Tech Debt):**
- Spinner resolution edge case: stillPending.add(id) missing — spinner always clears after one 5s poll regardless of PLC confirmation; unobservable in normal <1s ack scenario
- connectionId vs connectionNumber naming inconsistency across MongoDB → Vue → API body
- No VERIFICATION.md for Phase 4 (UAT.md covers functional testing)
- Nyquist VALIDATION.md draft across all phases (PoC has no automated test project)
- LCID hardcoded to 1033 (English only)
- No auto-resubscribe after alarm subscription failure mid-session
