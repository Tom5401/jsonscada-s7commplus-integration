  <action>
Create `PushValueDialog.vue` as a Vue 3 `<script setup>` SFC with Vuetify components. The dialog receives a leaf node item and lets the user push a new value to the PLC via OPC WriteRequest.

**Props/emits:**
- `modelValue` (Boolean) — controls dialog visibility via `v-model`
- `item` (Object) — the leaf node object (shape in interfaces block above; null when dialog closed)
- Emit `update:modelValue` to close

**Template structure (per D-01 dialog pattern from S7PlusAlarmsViewerPage.vue):**
```
v-dialog v-model="dialogOpen" max-width="450" persistent
  v-card
    v-card-title: "Modify — {{ item?.name }}"
    v-card-text:
      - Current value display: <code>{{ currentDisplayValue }}</code> (use formatLeafValue logic — digital shows TRUE/FALSE)
      - For digital tags (item.type === 'digital'): v-select with items ['TRUE', 'FALSE'], default to current boolean state
      - For non-digital tags: v-text-field v-model="inputValue", with the current value as initial
      - v-alert v-if="resultMsg" :type="resultType" density="compact" class="mt-3" — inline feedback
    v-card-actions:
      v-spacer
      v-btn @click="close" :disabled="sending" — "Cancel"
      v-btn color="primary" @click="submitWrite" :loading="sending" — "Write"
```

**OPC WriteRequest construction (per D-03, matching websage.js directCommandExec):**
- Inline the OPC constants (opc-codes.js is NOT an ES module):
  ```js
  const OPC = { WriteRequest: 671, WriteResponse: 674, Numeric: 0, Value: 13, Double: 11, String: 12, Namespace: 2, Good: 0 }
  ```
- `NodeId.Id` = `item.commandOfSupervised` (the paired command point's _id — NOT the display tag's _id)
- Digital tags: `Value: { Type: OPC.Double, Body: selectedValue === 'TRUE' ? 1.0 : 0.0 }`
- Analog tags: parse `inputValue` with `parseFloat()`. If `isNaN`, use `{ Type: OPC.String, Body: inputValue }`. Otherwise `{ Type: OPC.Double, Body: parseFloat(inputValue) }`
- POST to `/Invoke/` (NOT `/Invoke/auth/`) with `Content-Type: application/json`

**Response validation (per websage.js pattern):**
- Check: `data.ServiceId === OPC.WriteResponse && data.Body?.ResponseHeader?.ServiceResult === OPC.Good && data.Body?.Results?.[0] === OPC.Good`
- On success (per D-02): set `resultType = 'success'`, `resultMsg = 'Value written successfully'`. Start a `setTimeout` of 2000ms that calls `close()` to auto-close.
- On failure (per D-02): set `resultType = 'error'`, `resultMsg` = server message from `data.Body?.ResponseHeader?.StringTable?.[2] || 'Write failed'`. Do NOT close.
- On fetch error (network): set `resultType = 'error'`, `resultMsg = 'Network error'`. Do NOT close.

**close() function:** Reset `resultMsg`, `sending`, `inputValue`. Emit `update:modelValue` with `false`.

**Watch `modelValue`:** When dialog opens (true), initialize `inputValue` from `item.value` (or `item.value ? 'TRUE' : 'FALSE'` for digital), clear `resultMsg`.
