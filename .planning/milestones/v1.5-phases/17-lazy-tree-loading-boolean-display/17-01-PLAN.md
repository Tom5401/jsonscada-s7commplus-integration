  Rewrite TagTreeBrowserPage.vue. The component currently fetches the ENTIRE tag subtree via `listS7PlusTagsForDb` and builds a full in-memory tree. Replace this with lazy one-level-at-a-time loading using the Phase 16 `listS7PlusChildNodes` endpoint.

  **Data model change:**
  Each tree node becomes:
  ```javascript
  {
    id: doc.ungroupedDescription,           // full path, e.g. "MyDB.Struct1.Tag1"
    name: lastSegmentOf(doc.ungroupedDescription), // display name, e.g. "Tag1"
    children: doc.hasChildren ? [] : undefined,    // [] = expandable, undefined = leaf
    isLeaf: !doc.hasChildren,
    // Leaf-only fields:
    type: doc.type,                         // "digital", "analog", "string", "json"
    value: doc.value,
    valueString: doc.valueString,
    address: doc.protocolSourceObjectAddress,
    commandOfSupervised: doc.commandOfSupervised,
  }
  ```

  **Helper function `mapDocToNode(doc)`:**
  Extract the last segment of `doc.ungroupedDescription` by splitting on `'.'` and taking the last element for the display name. Set `children: []` when `doc.hasChildren === true` (folder, expandable). Set `children: undefined` when `doc.hasChildren === false` (leaf node, no expand arrow).

  **Replace `loadTree()` with `loadRootChildren()`:**
  On mount, call `fetch('/Invoke/auth/listS7PlusChildNodes?connectionNumber=${connectionNumber.value}&path=${encodeURIComponent(dbName.value)}')`. Map response docs to tree nodes via `mapDocToNode`. Create root item `{ id: dbName.value, name: dbName.value, isLeaf: false, children: mappedNodes }`. Set `treeItems.value = [root]`. Set `openedNodes.value = [dbName.value]` to auto-expand root (preserves current UX from v1.4).

  **Add `onLoadChildren(item)` callback for Vuetify `load-children`:**
  This async function receives the tree item being expanded. Call `fetch('/Invoke/auth/listS7PlusChildNodes?connectionNumber=${connectionNumber.value}&path=${encodeURIComponent(item.id)}')`. Map response to nodes via `mapDocToNode`. Set `item.children = mappedNodes`. This is called exactly once per node — Vuetify caches after first expand. If fetch returns empty array, set `item.children = []` (renders as empty folder, no infinite spinner).

  **Template: Add `:load-children="onLoadChildren"` to `<v-treeview>`:**
  The v-treeview tag becomes:
  ```html
  <v-treeview
    :items="treeItems"
    item-value="id"
    item-title="name"
    item-children="children"
    :load-children="onLoadChildren"
    v-model:opened="openedNodes"
    open-strategy="multiple"
    density="compact"
    class="elevation-1"
  >
  ```

  **Replace `refreshValues()` with scoped refresh:**
  Remove the old `refreshValues` that calls `listS7PlusTagsForDb` (full subtree fetch). New `refreshValues`:
  1. Walk the tree from root collecting all expanded parent paths — a parent path is "expanded" if its `id` appears in `openedNodes.value` AND it has loaded children (children is a non-empty array of actual nodes, not the initial empty `[]`).
  2. For each expanded parent path, call `listS7PlusChildNodes` in parallel via `Promise.all`.
  3. For each response, patch matching leaf nodes' `value` and `valueString` in-place (find by `id` match). Do NOT replace `item.children` arrays — that would lose expand state. Instead iterate response docs and for each doc, find the matching child node in the parent's children array by `id === doc.ungroupedDescription` and update `node.value = doc.value; node.valueString = doc.valueString`.
  4. After patching, call `touchExpandedLeafTags()` to keep TTL alive.

  **Helper function `getExpandedParentPaths()`:**
  Walk the tree recursively. For each node: if `openedNodes.value` includes `node.id` AND `Array.isArray(node.children)` AND `node.children.length > 0` AND at least one child has actual data (not the initial empty-array placeholder), add `node.id` to the set of expanded paths. Return array of these paths.

  **Modify `getExpandedLeafTags()` for new data model:**
  The logic is the same — walk tree, collect leaf nodes under expanded parents. But now check `node.children === undefined` (leaf) instead of `node.isLeaf` being a separate flag. Actually keep `node.isLeaf` as set above for clarity. The function collects `{ connectionNumber, protocolSourceObjectAddress: node.address }` for each leaf under an expanded parent. Guard: if result is empty, skip the touch call (backend rejects empty array with 400).

  **Add `formatLeafValue(item)` for boolean display (DISP-01):**
  ```javascript
  function formatLeafValue(item) {
    if (item.type === 'digital') {
      return item.value ? 'TRUE' : 'FALSE'
    }
    return item.value
  }
  ```
  Note: the `type` field in realtimeData is `"digital"` for Bool/BBool tags (set by `GetTypeFromAsdu` in the C# driver). There is no `protocolSourceDataType` field — the ROADMAP success criterion references `protocolSourceDataType` but the actual MongoDB field is `type`, which is already returned by the endpoint.

  **Template: Use `formatLeafValue` in append slot:**
  Replace `{{ item.value }}` with `{{ formatLeafValue(item) }}` in the `<code>` element inside the `#append` template slot. Keep the existing `v-chip` for type and `span` for address unchanged.

  **Remove dead code:**
  Delete `buildTree()` function entirely — no longer needed (was building full in-memory tree from flat docs).
  Delete `patchLeafValues()` function entirely — replaced by scoped in-place patching in `refreshValues`.
